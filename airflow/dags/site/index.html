Legend

  • Median / mean gap – minutes between buses compared with the target headway.
  • Std dev – how much the gaps fluctuate; higher = more uneven service.
  • Health score (0–1+) – 1.0+ is very regular, values closer to 0 mean bunching or big gaps.

Colour key

Green – very reliable / mostly regular service.
Amber – uneven gaps; watch for bunching.
Red – high bunching risk or very uneven waits.
Grey – not enough data yet for this route/direction.

How this MVP works

  1. An Airflow DAG hits the MBTA /vehicles API every N minutes and saves raw snapshots (bronze).
  2. A transform flattens each snapshot into a vehicles_*.csv file (silver).
  3. A headway engine computes gaps per route_id, direction_id pair and produces a Headway Health Score (gold).
  4. A small sync script copies the latest CSVs into site/data, and this static UI reads them directly in your browser.
